### Issues Encountered & Lessons
- **1GB RAM caused OOM on every OpenClaw CLI command** (onboard, doctor, config set). Resized to 4GB which solved everything. Minimum 2GB for OpenClaw, 4GB recommended.
- `gateway.bind: "0.0.0.0"` is NOT a valid config value -- must use `"loopback"` (or `"tailnet"`)
- When config is in invalid state, `openclaw config set` refuses to work. Fix: edit JSON directly with `sed`.
- `openclaw auth add --provider openrouter` doesn't exist in this version
- `auth.profiles.openrouter:default.apiKey` can't be set via `config set` -- API keys are handled during onboarding or via environment variables
- Swap (2GB) was added early but wasn't the real fix -- Node.js heap limit was the bottleneck
- DigitalOcean web console works fine for full setup (SSH from Mac timed out, probably hotel network)
- 0x0.st is a great quick file transfer when SSH isn't available between machines -- BUT files are publicly accessible to anyone with the URL. Not ideal for personal data. Use SSH or private Git repo going forward.
