Continued from morning. Focus: nav menu sizing, font scale, marquee heading.

### What got done:

**Nav menu:**
- type_navigation_size set to 20 via settings_data.json (theme's native setting, clean)

**Sweet Treat marquee heading:**
- Heading "sweet treat collection" removed entirely from marquee (Sunshine preferred no heading above products)
- Was causing confusion -- heading was being overridden by body rules repeatedly

**Font scaling attempt (rocky session):**
- Tried Summer Fridays type scale (body 16px, h2 26px) -- Sunshine didn't like it
- Tried +9px via calc(1em + 9px) on named elements -- cart badge blew up (span selector was too broad)
- Tried html root font-size approach multiple times:
  - +3px, +6px, +9px increments via calc(62.5% + Xpx) -- caused button/icon scaling issues
  - Final: `html { font-size: 18px !important }` (making 1rem = 18px instead of 16px)
- Cart count bubble pinned to 16x16px fixed to prevent scaling
- Header icons (search/account/cart svg) pinned to 20px fixed

**Current state of custom.css (as of ~3pm Mar 14):**
- Root font-size: 18px (sitewide rem scale up)
- Header icons: 20px fixed
- Cart bubble: 16px fixed, 9px font
- Nav menu items: 20px (via settings_data, not custom.css)
- All font-size TYPE SYSTEM overrides stripped -- using Be Yours defaults scaled by 18px root
- text-transform: lowercase still active sitewide
- letter-spacing: -0.02em still active on named elements
- Marquee heading: REMOVED

**Key lesson learned today:**
- DON'T use `span`, `a`, `div` in font-size rules -- catches icons, badges, cart counts
- DON'T use html root font-size scaling unless you also explicitly pin all icons/badges to px
- Buttons should always be set via `var(--font-button-size)` not overridden with px
- `calc(1em + Xpx)` approach sounds clever but `1em` inherits differently per context -- not reliable
- Safest approach for uniform text scale: change root font-size AND pin exceptions (icons, badges, buttons) to fixed px

**Sunshine's frustration level:** high by ~2:45pm. Multiple reversions. She took a break. Be careful not to break things trying to "improve" them. When she says revert, revert cleanly and don't experiment further without clear direction.
